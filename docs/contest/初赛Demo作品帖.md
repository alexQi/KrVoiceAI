# 【生活娱乐】EnlyAI · 虚拟人口播智能体 — 10分钟生成商用级口播视频

> 一句话介绍：输入文案，自动完成"文案润色→声音克隆→数字人唇形同步→字幕→BGM→封面→多平台发布"全流程，本地运行、隐私可控、零云端成本。

## 一、创意介绍

做一条口播短视频，要写文案、录音、拍摄、剪辑、加字幕、配 BGM、做封面、再发到抖音/B站/快手/视频号——整个流程至少涉及 6-8 个工具，门槛高、耗时长。更头疼的是，数字人口播视频的"嘴唇和声音不同步"几乎是行业通病。

**EnlyAI 解决方案**：一款本地运行的口播视频自动化生成系统，输入一段文案，10 分钟自动产出一条商用级口播视频，唇形同步精度对标旗博士，但完全本地、隐私可控、零云端成本。

## 三、核心功能展示

### 1. 文案提取（链接→文案，3秒完成）
- 粘贴抖音分享链接 → 自动提取口播文案
- 支持 抖音/快手/B站/YouTube
- 24小时URL缓存，避免重复提取
- **修复前**：3分钟 → **修复后**：3秒

### 2. 数字人唇形同步（核心卖点）
- Wav2Lip 视频驱动模式：保留原视频头部动作+表情，只换嘴型
- GPU加速：MX450 显卡 56s 音频 29 分钟完成（CPU 需 60+ 分钟）
- 质量优先：resize_factor=1 原分辨率推理 + wav2lip_gan.pth + nosmooth 平滑窗口
- **唇形同步达商用标准**，无"声音提前嘴滞后"问题

### 3. 字幕生成（词级时间戳逐字高亮）
- faster-whisper 本地转写，词级时间戳
- 卡拉OK逐字高亮效果（对标剪映/抖音）
- 字号 56 + 描边 3，长句自动折行
- 字幕与人声同步（adelay 延迟补偿）

### 4. BGM 混音 + 画中画 B-roll
- 6 首预置 BGM，音量 0.25 + amix 混音（不盖过人声）
- 画中画 B-roll：支持图片/视频，按字幕段落选择插入位置
- 转场效果：fade/slide/zoom

### 5. 多平台一键发布
- **B站**：API 直传（bilibili-api-python 库）
- **抖音/快手/视频号**：Playwright 浏览器自动化
- **傻瓜化登录**：B站扫码登录、其他平台浏览器自动提取 Cookie
- 一次配置，4 平台同步发布

## 四、技术实现

| 模块 | 技术方案 | 亮点 |
|------|---------|------|
| 文案提取 | Playwright + yt-dlp + faster-whisper | 网页抓取+视频下载+ASR三级降级，3秒完成 |
| 文案润色 | LLM (DeepSeek/MiMo) | 原创度检测，避免平台查重 |
| 声音克隆 | MOSS-TTS-Nano (0.1B ONNX) | 纯 CPU，5 秒样本零样本克隆 |
| 数字人 | Wav2Lip + CUDA GPU加速 | 视频驱动保留原动作，唇形同步商用级 |
| 字幕 | faster-whisper + ASS | 词级时间戳，卡拉OK逐字高亮 |
| BGM | FFmpeg amix | 音量 0.25，不盖过人声 |
| B-roll | FFmpeg overlay | 画中画，按字幕段落插入 |
| 发布 | bilibili-api + Playwright | 4平台自动化，扫码登录傻瓜化 |
| 编排 | SQLite 状态机 | 断点续跑 + 指数退避重试 |
| UI | FastAPI Web | 苹果风格设计，7标签页，26个API端点 |

**完整流水线**：
```
script_extract → script_write → originality_check → tts → avatar → subtitle → broll → title → cover → compose → publish
```

## 五、使用 TRAE 的开发过程

本项目全程使用 **TRAE IDE** 开发，TRAE 在以下环节发挥了关键作用：

1. **架构设计**：通过 TRAE AI 助手梳理论创口播视频全流程，设计了 11 步流水线架构
2. **代码生成**：9 大模块的核心代码由 TRAE 辅助生成，包括 Wav2Lip 集成、FFmpeg 命令编排、Playwright 自动化等
3. **Bug 修复**：本次 E2E 测试中发现的 4 个核心 Bug 全部由 TRAE 辅助定位和修复：
   - 音色试听 30s 延迟 → 改用轻量 API 端点，3.5s 完成
   - B-roll 弹窗无叉号 → 统一所有弹出框交互
   - 文案提取 3 分钟 → 优化 Playwright 等待策略 + 缓存
   - GPU 加速无效 → 修复 CUDA 检测逻辑 + 安装 CUDA 版 PyTorch
4. **E2E 测试**：TRAE 辅助编写端到端测试脚本，完整验证从文案提取到视频生成的全流程
5. **竞品分析**：通过 TRAE 调研闪剪AI、即灵AI等竞品，确定差异化优势
