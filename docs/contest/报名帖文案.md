# 【生活娱乐】EnlyAI · 虚拟人口播智能体

---

## 1. 创意名称 + 创意介绍

**创意名称**：EnlyAI · 虚拟人口播智能体

**想解决什么问题**：
做一条口播短视频，要写文案、录音、拍摄、剪辑、加字幕、配 BGM、做封面、再发到抖音/B站/快手/视频号——整个流程至少涉及 6-8 个工具，门槛高、耗时长，普通创作者很难一个人完成。现有云端方案还要把内容上传服务器，存在隐私、成本和网络依赖问题。

**为什么会想到做这个**：
我自己就是短视频创作者，每次做一条 1 分钟的口播视频要花 2-3 小时在重复的剪辑/对轨/配字幕上。更头疼的是，数字人口播视频的"嘴唇和声音不同步"几乎是行业通病——声音播了 1 秒嘴才动，一看就很假。我想做一个本地可运行、全流程一键化、且把唇形同步做到可商用标准的工具，让自己和更多普通人 10 分钟就能产出一-条高质量口播视频。

**大概是什么产品**：
一款本地运行的口播视频自动化生成系统（桌面软件/Web UI），输入一段文案，自动完成"文案润色 → 本地声音克隆 → Wav2Lip 数字人唇形同步 → 字幕生成（词级时间戳逐字高亮）→ BGM 混音 → 封面生成 → 多平台一键发布"全流程，对标旗博士但完全本地、隐私可控、零云端成本。

## 2. 目标用户及痛点

**面向哪些用户**：
- 短视频口播创作者（知识科普、情感故事、资讯解读、产品带货等赛道）
- 自媒体运营者、个人 IP 打造者
- 中小企业市场/新媒体岗位，需要批量产出口播内容
- 不想露脸但想做口播视频的"声优型"创作者

**在什么场景下使用**：
- 日常口播视频生产：写好文案 → 一键生成成片 → 直接发布
- 批量矩阵号运营：一次配置形象和音色，多条文案批量产出
- 数字人直播素材预制：提前生成口播片段用于直播插播
- 知识科普视频制作：文案驱动，自动加字幕和 BGM

**当前痛点**：
- **流程碎片化**：文案、录音、剪辑、字幕、发布要用 5-8 个不同工具，切换成本高
- **数字人嘴唇不同步**：市面数字人口播视频"声音提前嘴滞后"是通病，一眼假，影响完播率
- **云端方案贵且不安全**：旗博士等 SaaS 按条收费，内容上传服务器，隐私和商业机密泄露风险
- **真人出镜门槛高**：不想露脸的创作者（声优、社恐、形象焦虑）无法做口播
- **发布重复劳动**：一条视频要分别登录抖音、B站、快手、视频号手动上传，重复 4 次

## 3. 价值与意义

**商业价值**：
- **降本**：本地运行零云端成本，对标旗博士按条收费模式，长期使用可省 90%+ 成本
- **增效**：把 2-3 小时的视频制作流程压缩到 10 分钟内，产能提升 10 倍以上
- **变现**：可服务中小商家做口播带货、矩阵号运营、代运营服务，形成商业化路径

**效率提升**：
- **全流程一键化**：从文案到成片到发布，一个系统搞定，告别 8 个工具来回切换
- **傻瓜化登录**：B站扫码登录、抖音/快手浏览器自动提取 Cookie，用户无需任何专业操作
- **断点续跑**：SQLite 状态机 + 指数退避重试，长任务中断也能恢复，不浪费算力

**社会价值**：
- **降低创作门槛**：让不想露脸、不会剪辑、没有专业设备的普通人也能做口播视频，促进内容创作普惠
- **隐私可控**：本地处理不上传云端，保护创作者声音/形象等生物特征数据
- **无障碍传播**：自动字幕让听障用户也能消费视频内容

## 附：技术亮点（供评审参考）

| 模块 | 技术方案 | 亮点 |
|------|---------|------|
| 声音克隆 | MOSS-TTS-Nano (0.1B ONNX) | 纯 CPU，5 秒样本零样本克隆，无需云端 |
| 数字人 | Wav2Lip 视频驱动 | 羽化嘴部聚焦融合 + 居中平滑窗口，唇形同步达商用标准 |
| 字幕 | faster-whisper + ASS | 词级时间戳，卡拉OK逐字高亮，长句自动折行 |
| 发布 | bilibili-api + Playwright | B站API直传 + 抖音/快手浏览器自动化，扫码登录傻瓜化 |
| 编排 | SQLite 状态机 | 断点续跑 + 指数退避重试 |
| UI | FastAPI Web + Gradio | 7 标签页，26 个 API 端点 |

- ✅ BGM 可听且不盖过人声（音量 0.25 + amix 混音）

- ✅ 字幕长句自动折行（按标点 + 字数）
- ✅ 字幕字号 56 + 描边 3，抖音爆款风格
- ✅ 一键发布全链路验证（Cookie管理 + manifest生成 + 4平台状态正确）
